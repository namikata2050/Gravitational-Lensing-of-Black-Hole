# ブラックホール重力レンズシミュレーター (Event Horizon)

一般相対性理論に基づき、ブラックホールの質量による空間のゆがみが光を曲げる「重力レンズ効果」をブラウザ上でシミュレーションするアプリケーションです。
物理演算・数値計算部分をC++で記述し、Emscriptenを用いてWebAssembly（Wasm）へコンパイルすることで、高速かつリアルタイムなシミュレーションを実現しています。

## 🌐 オンラインデモ (Online Demo)
インストール不要で、今すぐブラウザ上で体験することができます。
👉 **[シミュレーターを体験する（GitHub Pages）](https://namikata2050.github.io/Gravitational-Lensing-of-Black-Hole/)**

### 🎮 操作方法
*   **カメラの回転**: 画面上を「左クリックしながらドラッグ」することで、視点を動かして周囲の星空を見渡せます。
*   **質量の変更**: 左側パネルの **Mass** を選ぶと、ブラックホールの強さを変更できます（変更後、光線の再計算が自動で走ります）。
*   **距離の変更 (ズーム)**: **Distance** のスライダーを動かすと、ブラックホールとの距離を調整できます。
*   **背景画像のアップロード**: **Upload Background** から好きな画像を選択して、ブラックホールの背景に適用できます。


## 概要 (Overview)
アインシュタインの一般相対性理論によれば、ブラックホールの巨大な質量は周囲の時空を極限まで歪めます。そこを通る光は歪んだ空間に沿って進むため、進路が曲がります。この現象を**重力レンズ効果**と呼びます。

本プロジェクトは、この現象を一般相対性理論のシュワルツシルト解（自転しないブラックホールモデル）に基づき正確にシミュレーションします。

### 主な機能
1. **高精度な物理演算**:
   4次のルンゲ・クッタ法（微分方程式の解法）を用いて、光子（光の粒）の軌道を時間ステップごとに追跡（レイトレーシング）します。
2. **高速リアルタイム描画**:
   重い物理シミュレーションを事前計算マップ（Distortion Map）として非同期で作成。カメラの回転や拡大縮小の際にはそのマップを高速参照することで、ブラウザ上でも遅延なく滑らかな操作が可能です。
3. **対話型ユーザーインターフェース**:
   - **質量（シュワルツシルト半径）調整**: ブラックホールの大きさを変更できます。
   - **ズーム（カメラ距離）調整**: ブラックホールへの距離を変更できます。
   - **背景画像のアップロード**: 宇宙だけでなく、好きな画像をブラックホールの背景に配置して歪みを確認できます。
   - **PIP（ピクチャー・イン・ピクチャー）**: 画面右上に「重力がない場合（歪んでいない本来の背景）」を同時に表示し、どれだけ歪んでいるかを比較できます。

## コード構造 (Code Structure)

ソースコードは `src` ディレクトリに配置されています。

*   **[src/vec3.h](file:///c:/Users/hiroto yamada/Programming/C++/BH/src/vec3.h) (3次元ベクトルクラス)**
    *   3次元空間の座標や進む向き、速度を表す基本クラス `Vec3` を定義しています。ベクトルの加減乗除や長さの計算、内積、外積といった3次元幾何学に必要な演算が実装されています。
*   **[src/physics.h](file:///c:/Users/hiroto yamada/Programming/C++/BH/src/physics.h) (シュワルツシルト時空の物理方程式)**
    *   光子の位置と運動量（進む向き）を管理する `PhotonState` 構造体と、一般相対性理論に準拠したシュワルツシルトブラックホール周辺での光の運動方程式を定義しています。
*   **[src/ode_solver.h](file:///c:/Users/hiroto yamada/Programming/C++/BH/src/ode_solver.h) (常微分方程式ソルバー)**
    *   微分方程式をコンピュータで解くための最も代表的なアルゴリズムである**「4次のルンゲ・クッタ法」** (`rk4_step`) を提供します。
*   **[src/main.cpp](file:///c:/Users/hiroto yamada/Programming/C++/BH/src/main.cpp) (メインアプリケーション＆描画ループ)**
    *   SDL2を用いた描画および入力処理を制御します。
    *   ブラックホールの質量などの変更命令を受けると、バックグラウンドで段階的に光線の軌跡を事前計算 (`distortion_map`) します。
    *   JavaScriptとのインターフェース（`EMSCRIPTEN_BINDINGS`）を提供し、質量や背景画像のデータの送受信を行います。
*   **[src/shell.html](file:///c:/Users/hiroto yamada/Programming/C++/BH/src/shell.html) (フロントエンド HTML/JS)**
    *   操作パネルやローディングゲージ、シミュレーターの描画キャンバス（Canvas）を管理するUIファイルです。
    *   JavaScriptで読み込んだ画像データを、C++側のヒープメモリを直接確保・解放する手法（`_malloc`/`_free`）を用いて高速にWasm側に転送する仕組みを持っています。

## 使用している技術 (Tech Stack)
*   **C++17**: シミュレーションロジックおよび高速画像書き込み処理
*   **Emscripten / WebAssembly**: C++コードのWebブラウザ向けビルドとJavaScript双方向連携
*   **SDL2 (Simple DirectMedia Layer)**: ピクセル描画バッファの更新・制御およびマウス入力処理
*   **HTML5 / CSS3 / JavaScript (ES6)**: UI（ユーザーインターフェース）デザインおよび画像読み込み処理

## ビルドおよび実行方法 (Build & Run)

### 前提条件
*   [Emscripten SDK (emsdk)](https://emscripten.org/docs/getting_started/downloads.html) がインストールされ、パスが通っていること。

### ビルド手順
Windows環境の場合、ルートディレクトリにある `build.bat` を実行することでコンパイルが可能です。

1.  コマンドプロンプトやPowerShellを開きます。
2.  ルートディレクトリへ移動し、`build.bat` を実行します。
    ```cmd
    build.bat
    ```
    ※ `build.bat` 内部では、`emsdk_env.bat` のパスを呼び出してコンパイル環境を整えた後、以下の `emcc` コマンドを実行して `build` フォルダ内に出力ファイルを書き出します。

    **コンパイルコマンド詳細:**
    ```bash
    emcc src/main.cpp -o build/bh_sim.html -s USE_SDL=2 --bind --shell-file src/shell.html -s "EXPORTED_FUNCTIONS=['_malloc', '_free', '_main']" -s "EXPORTED_RUNTIME_METHODS=['HEAPU8']" -s ALLOW_MEMORY_GROWTH=1
    ```

### 実行方法
ビルドが成功すると、`build` ディレクトリに以下のファイルが生成されます。
*   `bh_sim.html`（本体画面）
*   `bh_sim.js`（Wasm呼び出し用スクリプト）
*   `bh_sim.wasm`（ビルドされたバイナリ）

ブラウザのセキュリティ制限（CORSポリシーなど）により、ローカルファイルを直接ダブルクリックで開くと正しく動作しない場合があります。
ローカルWebサーバーを立ち上げて動作させてください。

**例（Pythonを使用する場合）:**
```bash
python -m http.server 8000
```
その後、ブラウザで `http://localhost:8000/build/bh_sim.html` にアクセスしてください。
